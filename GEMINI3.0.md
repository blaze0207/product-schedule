# 撚二科生產戰情室 3.0 技術手冊 (GEMINI3.0.md)

本文件定義 3.0 「專業事實版 v3.2 Professional」架構。此版本徹底解耦計畫與現場，整合工程師責任區篩選、POY 智慧清洗對接、全場耗用分析、產量結算指標以及全平台響應式 (RWD) 視覺體驗。

## 1. 系統哲學 (System Philosophy)
*   **事實為本**：以現場錄入的生產資訊表為唯一數據來源，消滅計畫與現實的資訊落差。
*   **精準結算**：採「結算日（最新紀錄之前一天）」產量數據，確保日產指標之完整性與參考價值。
*   **管理分流**：新增工程師快捷篩選按鈕（羅工/吳工），提供專注的巡檢視野。
*   **專業視覺化**：採用深色主題、呼吸燈動畫與色彩警示系統，符合現代監控中心標準。

## 2. 核心運算邏輯

### A. 數據結算指標 (Settled Metrics)
1.  **結算日產量**：自動選取最新錄入日期的「前一作業日」作為基準，加總全場 Index 47 (當日生產T/台)。
2.  **本月累計繳庫**：計算 LISA DTY 庫存 (總庫存 - 外購)。
3.  **生產資訊日期**：顯示最新的一筆紀錄日期，代表數據同步的終點。

### B. POY 智慧對接與分析 (POY Intelligence)
1.  **倍數清洗**：自動移除批號末尾標記（如 `×2`, `*2`），產出 `clean_id` 用於數據對接與全場分析。
2.  **模糊匹配**：搜尋框支援部分批號輸入（如 82167），系統自動關聯匹配的 POY 深度分析卡片。
3.  **支撐預測**：`預估天數 = POY 總庫存 (A+A2) / (共用機台平均日產總量(T) * 1000)`。

### C. 工程師責任區定義 (Engineer Filters)
*   **羅工**：A07-09, A13-15, A19-21, J07, T04-06。
*   **吳工**：A10-12, A16-18, A22-24, J08, T01-03。
*   **篩選互動**：點擊工程師按鈕時自動重置 POY 搜尋條件，確保視野單一化。

## 3. UI 視覺與互動規範

### A. 視覺標準
*   **主題**：深色 Header (Navy Blue) 搭配現代背景 (Slate Grey)。
*   **狀態指示**：
    *   **運行中**：綠色脈衝呼吸燈。
    *   **停機**：指示燈與文字全面標紅並加粗。
*   **數據格式**：繳庫總量採 `總:{值}`，生產天數採純數字。

### B. 互動特色
*   **搜尋互動**：輸入框內置「加粗紅色 X」清除按鈕。
*   **點擊跳轉**：POY 批號可點擊，觸發分析並自動跳轉至頁面頂部。
*   **快捷篩選**：頂部按鈕組實現全部/羅工/吳工視角切換。

### C. 響應式標準 (RWD)
*   **Desktop**：高對比度專業表格視圖。
*   **Mobile (< 768px)**：表格自動轉換為數據卡片，利用 `data-label` 動態標註欄位名稱。

## 4. 檔案結構
*   `reality_analyzer.py`：後端數據分析、清洗與結算日產量引擎。
*   `generate_dashboard_v3.py`：專業版渲染器，負責互動 JS、RWD CSS 與按鈕篩選邏輯。
*   `production_dashboard.html`：最終戰情室成品檔案。
