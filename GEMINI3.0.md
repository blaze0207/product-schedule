# 撚二科生產戰情室 3.0 技術手冊 (GEMINI3.0.md)

本文件定義 3.0 「專業事實版 v3.2 Professional」架構。此版本徹底解耦計畫與現場，整合 POY 智慧清洗對接、全場耗用分析、模糊搜尋、今日產量指標以及全平台響應式 (RWD) 視覺體驗。

## 1. 系統哲學 (System Philosophy)
*   **事實為本**：以現場生產資訊表為唯一數據來源，消滅計畫與現實的落差。
*   **即時監控**：不僅顯示任務，更提供「今日生產總量」與「全場支撐天數」輔助管理。
*   **視覺警示化**：運用色彩、動畫（呼吸燈）與紅色加粗標註，第一時間呈現停機與低庫存風險。
*   **全平台覆蓋**：完美支援大螢幕看板與手機端即時觀測 (RWD)。

## 2. 核心運算邏輯

### A. 數據指標 (Key Metrics)
1.  **今日生產總量**：自動篩選《生產資訊表》最新日期，加總全場 Index 47 (當日生產T/台)。
2.  **本月累計繳庫**：計算 LISA DTY 庫存 (總庫存 - 外購)。
3.  **全場 POY 分析**：以 `clean_id` 彙總全場共用機台之「平均日產 (T)」，結合庫存預測支撐天數。

### B. 數據清洗與對接 (Data Normalization)
1.  **POY 智慧清洗**：自動移除批號末尾倍數標記（如 `×2`, `*2`）。
2.  **模糊匹配**：搜尋框輸入部分批號（如 82167）自動關聯深度分析卡片。
3.  **日期提取**：同步顯示「同步時間」、「生產資訊日期」與「庫存基準日」。

## 3. UI 視覺與互動規範

### A. 專業化樣式標準
*   **主題**：專業深色 Header (Navy Blue) 搭配現代背景 (Slate Grey)。
*   **狀態指示**：
    *   **運行中**：綠色脈衝呼吸燈。
    *   **停機**：紅色指示燈、紅色加粗文字與紅色警示邊條。
*   **數據呈現**：
    *   繳庫總量格式：`總:{數值}`。
    *   生產天數：純數字顯示（不帶單位）。
*   **搜尋互動**：輸入框具備醒目「加粗紅色 X」清除鍵；POY 批號具備「點擊即分析」跳轉功能。

### B. 欄位定義 (Simplified Professional Columns)
*   **機台**：機台號與運作側邊。
*   **狀態**：動態燈與警示文字。
*   **批號 / 規格**：當前生產任務組合。
*   **POY庫存(A+A2)**：原絲資訊、有效庫存量及生產履歷。
*   **平均日產**：本月平均每台日產量 (T)。
*   **生產天數**：本月累計天數（純數字）。
*   **繳庫量**：LISA 庫存分類明細與 `總:{合計}`。

### C. 響應式標準 (RWD)
*   **Desktop**：標準高對比專業表格視圖。
*   **Mobile (< 768px)**：自動轉換為「獨立數據卡片」模式，動態生成欄位標籤，方便滑動閱覽。

## 4. 檔案結構
*   `reality_analyzer.py`：後端數據分析與今日產量運算引擎。
*   `generate_dashboard_v3.py`：專業版 UI 渲染、RWD 邏輯與互動系統。
*   `production_dashboard.html`：最終產出的專業戰情室。
