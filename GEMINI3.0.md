# 撚二科生產戰情室 3.0 技術手冊 (GEMINI3.0.md)

本文件定義 3.0 「專業事實版 v3.2 Professional」架構。此版本徹底解耦計畫與現場，整合 POY 智慧清洗對接、全場耗用分析、模糊搜尋、產量結算指標以及全平台響應式 (RWD) 視覺體驗。

## 1. 系統哲學 (System Philosophy)
*   **事實為本**：以現場錄入的生產資訊表為唯一數據來源，消滅計畫與現實的資訊落差。
*   **預測領先**：透過「全場預估支撐天數」將數據轉化為補料決策。
*   **專業視覺化**：採用深色主題、呼吸燈動畫與色彩警示系統，模擬專業數據監控中心。
*   **無縫兼容**：支援 PC 大螢幕看板與手機端 (RWD) 隨時查閱。

## 2. 核心運算邏輯

### A. 生產總量結算 (Settle Production)
1.  **邏輯**：為確保數據完整性，系統自動選取最新錄入日期的「前一作業日」作為結算基準。
2.  **公式**：篩選結算日資料，加總全場 Index 47 (當日生產T/台)。
3.  **呈現**：以 `生產總量 (YYYY/MM/DD)` 標籤顯示於頂部 KPI 卡片。

### B. POY 智慧對接與分析 (POY Intelligence)
1.  **智慧清洗**：自動移除批號末尾倍數標記（如 `×2`, `*2`, `x2`），產出 `clean_id` 用於數據對接。
2.  **模糊匹配**：前端搜尋支援部分批號輸入（如 82167），系統自動關聯匹配的 POY 深度分析。
3.  **支撐預測**：`預估天數 = POY 總庫存 (A+A2) / (相關機台平均日產總量(T) * 1000)`。
4.  **對接原則**：後端運算採 `clean_id` 彙總，前端顯示保留原始標記。

### C. 日期資訊系統 (Multi-Date System)
*   **VAR_TIME**：數據同步（程式執行）即時時間。
*   **VAR_PROD_DATE**：生產資訊表最新紀錄日。
*   **VAR_STOCK_DATE**：LISA 庫存表財務基準日。
*   **VAR_SETTLE_DATE**：產量結算日（最新紀錄日之前一天）。

## 3. UI 視覺與互動規範

### A. 視覺標籤與狀態
*   **運行中**：綠色脈衝呼吸燈指示。
*   **停機**：紅色指示燈、紅色加粗文字。
*   **繳庫量**：分級標籤 (A/AX/B/C) 搭配 `總:{數值}` 合計顯示。
*   **生產天數**：純數字顯示。

### B. 互動系統
*   **搜尋列**：支援模糊搜尋，具備醒目「加粗紅色 X」一鍵清除鍵。
*   **點擊跳轉**：表格內 POY 批號可點擊，自動執行深度分析並平滑跳轉至頂部。

### C. 響應式標準 (RWD)
*   **斷點**：768px。
*   **轉換**：手機端將表格行轉換為獨立數據卡片，利用 `data-label` 動態標註欄位名稱。

## 4. 檔案結構
*   `reality_analyzer.py`：後端數據分析、清洗與結算日產量引擎。
*   `generate_dashboard_v3.py`：專業版渲染器，負責互動 JS、RWD CSS 與多重 replace 替換。
*   `production_dashboard.html`：最終戰情室成品檔案。
