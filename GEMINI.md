# 撚二科產銷戰情室 2.0 技術手冊 (GEMINI.md)

本文件定義 2.0 版本之模組化架構與核心運算邏輯，作為系統重構與回復之最高準則。

## 1. 系統架構 (System Architecture)
系統採用「模組化資料引擎」設計，將數據讀取、清洗、關聯與呈現徹底解耦：
1.  **`MasterTableEngine` (母表引擎)**：讀取 DTY 計畫，執行 ffill 補全，產出「聚合任務」清單。
2.  **`ActualsEngine` (實績引擎)**：讀取現場生產資訊，執行側邊識別與歷史平均效率運算。
3.  **`InventoryEngine` (庫存引擎)**：讀取 LISA 與 POY 庫存，執行智慧分攤與序列解析。
4.  **`generate_dashboard.py` (呈現層)**：調用引擎數據，執行頂部彙總運算並產出響應式 UI。

## 2. 核心數據邏輯 (Core Logic)

### A. 側邊身分與狀態判定 (Side & Status Recognition)
判定來源：`*撚二科生產資訊.xlsx` 之 Index 1 (斷絲.IAD.效率)。
1.  **身分識別 (Marker)**：
    *   **L 欄 (Index 11)** 有值 -> 判定為 **SIDE_1** (對應計畫 A/L 側)。
    *   **M 欄 (Index 12)** 有值 -> 判定為 **SIDE_2** (對應計畫 B/R 側)。
2.  **狀態判定 (Double-Check)**：
    *   **優先級 1**：若 L/M 欄位內包含文字（如：停機），該側狀態立即判定為該文字，產量歸 0。
    *   **優先級 2**：若 L/M 為數字，則產量抓取 **AV 欄 (Index 47)**。
    *   **優先級 3**：若 AV 產量為 0，則參考 **AA 欄 (Index 26)** 之原因文字。

### B. 自產繳庫量計算 (Self-Production Formula)
頂部看板之「本月自產繳庫量」採用財務扣除法：
*   **公式**：`自產總重 = (LISA 總庫存 H 欄最後一列) - (LISA 外購 H 欄最後一列)`
*   **目的**：確保大數指標與財務現狀 100% 吻合，不受計畫對接率影響。

### C. 批號智慧對接 (Join Key Optimization)
*   **基礎清洗**：移除 `FD`, `FP`, `LIKE`, 空格。
*   **進階對接 (Join Key)**：使用正則 `(?<=\d)[A-Z]+$` 移除緊跟在數字後的結尾字母裝飾碼（如 `2F42S2K` -> `2F42S2`），保留核心編碼 `2FA042`。

## 3. UI 功能與 RWD 規範
1.  **跨機台 POY 彙總**：前端 JS 即時加總所有使用該 POY 的機台消耗量，算出「全場支撐天數」。
2.  **響應式卡片 (Mobile Card Mode)**：在手機端將表格轉化為 flex 卡片，內容包裹於 `.td-content` 中，標籤由 `::before` 偽元素生成，確保顯示不空白。
3.  **達標動態標籤**：
    *   `demand < 0`：顯示「缺: [數量]」 (灰色)。
    *   `demand >= 0`：顯示「盈餘: [數量]」 (綠色)。
