# 撚二科產銷戰情室開發手冊 (GEMINI.md)

本文件定義本專案之核心邏輯、數據處理機制及自動化規範，確保系統之準確性與維護性。

## 1. 數據流與整合架構 (Data Flow)

系統整合三方數據源進行動態運算：
1. **排產計畫 (May表)**：`DTYMay26*.xlsx` 之第一分頁，定義生產「目標」。
2. **繳庫庫存 (庫表)**：`DTYMay26*.xlsx` 之第二分頁，定義「已完成實績」。
3. **現場實產 (實產表)**：`115年04月撚二科生產資訊.xlsx` 之 Index 1 分頁，定義「現場即時數據」。

## 2. 核心數據邏輯 (Core Logic)

### A. 側邊序列匹配規則 (Side Sequence Rule)
由於實產表不直接標註 A/B 側，系統依據「資料出現序位」進行動態判定：
1. **AFK機台 (A07-24)**：第 1 筆出現 = **A側**，第 2 筆出現 = **B側**。
2. **TMT/33J機台 (T01-06, J07-09)**：第 1 筆出現 = **L側**，第 2 筆出現 = **R側**。
3. **匹配機制**：讀取計畫表之 `Mark` 欄位 (A/B/L/R)，精確對接上述序位數據。這決定了「實際批號」與「現場狀態文字」的精確歸屬。

### B. 產量與庫存分攤機制 (Apportionment Logic)
解決機台分側計畫導致數據倍計之核心：
1. **產量均分**：`單行實際 T/D` = `該機台現場總 T/D (不分側)` / `該機台在計畫表中的總行數`。
2. **庫存分攤**：`單行已繳庫量` = `該批號總庫存` / `該批號在計畫中出現的總次數`。
3. **效益**：確保合計數據與現場回報 100% 吻合，並產出科學的完工推估日期。

### C. 機台名稱智慧顯示 (Smart UI Display)
1. **多側生產模式**：若實產表對該機台有 2 筆(含)以上數據，顯示為 `A19 (A)`、`A19 (B)`。
2. **全機模式**：若實產表對該機台僅有 1 筆數據，自動簡化顯示為 `A08`，不顯示標註，直觀反映一台生產狀態。

### D. 產量抓取與狀態文字
1. **抓取優先序**：1. Index 47 (實際產量T/日) -> 2. Index 26 (AA1 欄位) -> 3. 強制設為 0。
2. **狀態文字**：產量為 0 時，擷取上述欄位之非數值文字 (如：`● 改紡`、`● 了機`)，以紅色粗體標註於機台編號下方。

### E. 系統自動推估完工日 (Forecast)
* **計算公式**：`數據更新日 + (計畫欠量 / (分攤後的實際 T/D * 1000))`。
* **智慧警告**：
    * 繳庫 ≥ 目標：顯示「**達標**」(綠色)。
    * 推估日 > 7 天：顯示「**預計完工: MM/DD**」(橘色警告)。
    * 正常：顯示「**預計完工: MM/DD**」(綠色)。

## 3. 過濾與篩選規則 (Filtering)

### 機台排除邏輯
* 完全排除 `V`, `S2` 相關行。
* 完全排除 `M01-M08` 與 `S01-S02` 機台，不納入總產量計算。
* 排除靜態文字行 (如 `庫存不排產`, `待排產`)。

### 工程師篩選範圍
