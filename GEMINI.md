# 撚二科產銷戰情室技術手冊 (GEMINI.md)

本文件定義本系統之數據整合架構、核心運算邏輯及 UI 功能規範，作為日後優化與維護之最高準則。

## 1. 數據源架構 (Data Sources)

系統採動態偵測機制，自動讀取資料夾內最新的四類 Excel 檔案：
1.  **排產計畫 (DTY表)**：`DTY*.xlsx` 之第一分頁。定義計畫目標、POY 批號與機台。
2.  **繳庫庫存 (LISA表)**：優先讀取 `每日-最新庫存(DTY-LISA).xlsx`。定義已完成實績（公斤）。若檔案不存在，則回退讀取 `DTY*.xlsx` 之第二分頁。
3.  **現場實產 (實產表)**：`*撚二科生產資訊.xlsx` 之 Index 1。定義現場即時 T/D 與文字狀態。
4.  **原料庫存 (POY表)**：`絲八科-庫存表*.xlsx` 之最後一分頁。定義 POY A/A2 級剩餘量。

## 2. 數據前置處理 (Data Pre-processing)

*   **向下填補 (ffill)**：針對計畫表中合併儲存格的機台名稱與 POY 批號執行 `ffill`，確保每一行數據均擁有完整的關聯資訊。
*   **動態月份解析**：從 `DTY` 檔名識別英文月份（如 May -> 5月），自動更新網頁標題。
*   **批號清洗**：統一將批號轉為大寫並簡化 ID，去除 FD 或 K 等前綴尾碼以利精確匹配。

## 3. 核心數據邏輯 (Core Logic)

### A. 數據分攤機制 (Apportionment Logic) - 防止倍計
1.  **實際產量均分**：`單行實際 T/D` = `機台現場總 T/D` / `計畫表中該機台出現總行數`。
2.  **庫存實績分攤**：`單行已繳庫量` = `該批號總庫存` / `計畫中該批號出現總次數`。
*   **目的**：確保戰情室統計之合計數據與原始表單 100% 吻合。

### B. POY 原料處理與預警
1.  **接續批號識別**：支援 `>` 或 `＞` 符號拆分。例如 `FP82166R>82167R` 會被視為兩筆原料。
2.  **前綴智慧補全**：若接續批號省略 `FP`（如 `>82167R`），系統自動補全為 `FP82167` 進行對接。
3.  **可用庫存過濾**：計算可用量時，**強制排除等級為 "B"** 的部分。
4.  **預警公式**：`預估支撐天數` = `(批號1庫存 + 批號2庫存)` / (`單行實際 T/D` * 1000)。
    *   🔴 < 3天閃爍；🟠 3~5天警告。

### C. 側邊序列判定 (Side Rule)
依資料出現順序自動判定：
*   **AFK機台**：1=A側, 2=B側。
*   **TMT/33J機台**：1=L側, 2=R側。
*   若現場僅一筆資料，則採「全機模式」不顯示側邊標註。

## 4. UI 功能與看板規範

### A. 管理層異常看板
現場回報文字（T/D 為 0 時）自動分類為六大項：
*   **改紡、了機、停機、待料、清車、檢修**。
*   歸類後於網頁頂部顯示總台數統計。

### B. POY 跨機台彙總查詢 (Advanced Analytics)
*   **功能**：於搜尋框輸入 POY 局部批號。
*   **邏輯**：系統會自動找出所有使用中機台，並加總它們的 `KG/日` 消耗速度，除以該原料的`剩餘總量`，得出**全場性**的精確可用天數。

### C. 篩選與過濾
*   **機台排除**：完全排除 `V`, `S2` 相關行；排除 `M01-M08`, `S01-S02` 機台。
*   **轄區切換**：提供吳、羅兩位工程師的轄區機台快速過濾按鈕。
