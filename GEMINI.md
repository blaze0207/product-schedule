# 撚二科產銷戰情室開發手冊 (GEMINI.md)

本文件定義本專案之核心架構、數據處理機制及自動化規範，確保系統之準確性、前瞻性與易維護性。

## 1. 系統架構與數據源 (Data Architecture)

系統動態整合四方數據源進行即時運算：
1.  **排產計畫 (DTY表)**：讀取 `DTY*.xlsx` 第一分頁。定義生產目標、機台配置與 POY 批號。
2.  **繳庫庫存 (庫表)**：讀取 `DTY*.xlsx` 第二分頁。定義已完成之實績庫存量。
3.  **現場實產 (實產表)**：讀取 `*撚二科生產資訊.xlsx` 之 Index 1 分頁。定義現場即時產量與機台文字狀態。
4.  **原料庫存 (POY表)**：讀取 `絲八科-庫存表*.xlsx` 之最後一個分頁。定義原料剩餘量。

> **自動化規範**：所有檔案均採動態搜尋（`glob`），系統自動解析檔名月份（如 May -> 5月）並抓取最新日期檔案。

## 2. 核心數據處理邏輯 (Core Logic)

### A. 側邊序列匹配規則 (Side Sequence Rule)
實產表不直接標註 A/B 側，系統依據現場資料「出現序位」判定：
*   **AFK機台 (A07-24)**：第 1 筆 = **A側**，第 2 筆 = **B側**。
*   **TMT/33J機台 (T01-06, J07-09)**：第 1 筆 = **L側**，第 2 筆 = **R側**。
*   **匹配機制**：對接計畫表 `Mark` 欄位。若現場僅有 1 筆資料，則自動回歸「全機模式」，不顯示側邊標註。

### B. 數據分攤機制 (Apportionment Logic)
防止因機台分側計畫導致的數據倍計：
1.  **產量均分**：`單行實際 T/D` = `該機台現場總產量 (不分側)` / `該機台在計畫中出現的總行數`。
2.  **庫存分攤**：`單行已繳庫量` = `該批號總庫存` / `該批號在計畫中出現的總次數`。

### C. 原物料 (POY) 預警邏輯
1.  **等級過濾**：計算可用庫存時，自動**排除等級為 "B"** 的重量。
2.  **明細顯示**：於網頁顯示總可用量的同時，細分列出 **A** 與 **A2** 的各自剩餘量。
3.  **可用天數推估**：`可用天數` = `POY可用庫存` / (`單行實際 T/D` * 1000)。
4.  **警示標準**：
    *   🔴 **危險 (Danger)**：< 3 天（網頁閃爍提醒）。
    *   🟠 **注意 (Warning)**：3 ~ 5 天。

## 3. 管理看板與異常分類 (Management Dashboard)

### A. 異常原因分類
系統自動解析現場回報文字（產量為 0 時之備註），歸類為以下六大類：
*   **改紡**、**了機**、**停機**、**待料**、**清車**、**檢修**。
*   歸類後於網頁頂部顯示統計數量，供管理層快速掌握稼動狀態。

### B. 完工日推估 (Forecast)
*   **公式**：`數據更新日 + (計畫欠量 / (實際 T/D * 1000))`。
*   **智慧標籤**：
    *   繳庫 ≥ 目標：顯示「**達標**」(綠色)。
    *   推估日 > 7 天：顯示「預計完工: MM/DD」(橘色警告)。

## 4. 篩選與過濾規則 (Filtering)

*   **排除對象**：`V`, `S2` 相關行；機台 `M01-M08`, `S01-S02`；靜態文字行（如 `庫存不排產`）。
*   **工程師轄區**：
    *   **吳**：A10-12, A16-18, A22-24, T01-03, J08。
    *   **羅**：A7-9, A13-15, A19-21, T04-06, J07。
