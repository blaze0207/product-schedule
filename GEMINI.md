# 5月產銷表專案開發手冊 (GEMINI.md)

本文件紀錄本專案的核心處理邏輯與表格格式規範，以便後續維護與功能擴充。

## 1. 資料處理邏輯 (Data Processing)

### 批號匹配 (Batch Matching)
- **邏輯名稱**：`simplify_id`
- **規則**：
  - 必須移除所有以 `FD` 開頭的 4 碼前綴 (例如：`FD2F`, `FD2G`, `FD2H`)。
  - 統一轉為 **大寫** 並去除前後空格。
  - 範例：`FD2FA042` -> `A042`, `FD2G04X2` -> `04X2`。
- **目的**：確保「May (排產)」分頁與「庫 (庫存)」分頁的批號能準確比對，特別是 AX 等級的 FD2G 格式。

### 過濾規則
- **跳過機台**：`V`, `S2`, `M4 (CW)`, `機台`。
- **關鍵字過濾**：若機台名稱包含 `庫存不排產` 或 `待排產` 則不列入計算。
- **無效數據**：天數 (Days) 必須大於 0 且不為空值。

### 數值計算
- **排產總量 (Target)**：`天數 * T/D * 1000` (單位：KG)。
- **需求量 (Demand)**：`已繳庫量 - 排產總量`。
- **生產進度 (Progress)**：`(已繳庫量 / 排產總量) * 100`，上限為 100%。

## 2. 表格格式規範 (Table Format)

### 欄位順序 (Column Order)
表格必須嚴格按照以下順序排列：
1. **機台** (Machine)
2. **批號** (Batch)
3. **規格** (Spec)
4. **排產日期** (Date Range)
5. **天數** (Days)
6. **T/D**
7. **排產總量 (KG)** (Target)
8. **已繳庫 (KG)** (Stored) - *包含分級標籤*
9. **進度** (Progress) - *進度條顯示*
10. **需求量 (KG)** (Demand) - *正數綠色，負數紅色*
11. **POY批號** (POY Batch)
12. **POY規格** (POY Spec)

### 已繳庫 (KG) 顯示細節
- **首行**：顯示總繳庫量 (粗體)。
- **次行**：分級標籤顯示，包含 `A`, `AX`, `B`, `C` 及 `Other`。
- **顏色規範**：
  - A: 綠色 (#22c55e)
  - AX: 藍色 (#3b82f6)
  - B: 橙色 (#f59e0b)
  - C: 深灰色 (#64748b)
  - Other: 淺灰色 (#94a3b8)

## 3. 篩選器功能 (Engineer Filters)
- **吳 (wu)** 負責機台：A10-12, A16-18, A22-24, T01-03, J08。
- **羅 (lo)** 負責機台：A7-9, A13-15, A19-21, T04-06, J07。

## 4. 自動化流程
- **腳本**：`generate_dashboard.py` (負責資料清洗與 HTML 生成)。
- **執行檔**：`更新生產進度表.bat` (負責執行 Python 腳本並推送到 GitHub)。
