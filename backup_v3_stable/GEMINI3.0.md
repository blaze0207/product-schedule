# 撚二科生產戰情室 3.0 技術手冊 (GEMINI3.0.md)

本文件定義 3.0 「專業事實版 v3.3 Professional」架構。此版本徹底解耦計畫與現場，整合工程師責任區篩選、POY 智慧清洗對接、全場耗用分析、數值居中對齊優化、產量結算指標以及全平台響應式 (RWD) 視覺體驗。

## 1. 系統哲學 (System Philosophy)
*   **事實為本**：以現場錄入的生產資訊表為唯一數據來源，消滅計畫與現實的資訊落差。
*   **預測領先**：透過「全場預估支撐天數」將數據轉化為補料決策。
*   **視覺標準化**：採用專業深色主題、呼吸燈動畫與「對齊優化」排版，模擬專業數據監控中心。
*   **無縫兼容**：支援 PC 大螢幕看板與手機端 (RWD) 隨時查閱。

## 2. 核心運算邏輯

### A. 數據結算指標 (Settled Metrics)
1.  **每日產量**：自動選取最新錄入日期的「前一作業日」作為基準，加總全場當日生產 T。
2.  **本月累計繳庫**：計算 LISA DTY 庫存 (總庫存 - 外購)。
3.  **生產資訊日期**：顯示最新的一筆紀錄日期，代表數據同步的終點。

### B. POY 智慧對接與分析 (POY Intelligence)
1.  **智慧清洗**：自動移除批號末尾倍數標記（如 `×2`, `*2`），建立清洗後的 `clean_id` 作為對接與彙總主鍵。
2.  **模糊匹配**：搜尋框支援部分批號輸入（如 82167），系統自動關聯匹配的 POY 深度分析。
3.  **支撐預測**：`預估天數 = POY 總庫存 (A+A2) / (共用機台平均日產總量(T) * 1000)`。
4.  **警示閾值**：支撐天數 < 3 天時，觸發紅色高亮加粗警示。

### C. 工程師責任區 (Engineer Filters)
*   **羅工**：A07-09, A13-15, A19-21, J07, T04-06。
*   **吳工**：A10-12, A16-18, A22-24, J08, T01-03。

## 3. UI 視覺與互動規範

### A. 對齊與排版優化 (Alignment Standards)
*   **靠左對齊**：機台、狀態、批號/規格、POY 庫存（適合複合與變動文字）。
*   **居中對齊**：預估支撐、平均日產、生產天數（數值欄位居中方便垂直比較）。
*   **垂直居中**：所有單元格採用 `vertical-align: middle`，確保多 POY 情況下重心一致。

### B. 視覺指示與格式
*   **狀態指示**：運行中（綠色呼吸燈）；停機（紅色指示燈與文字）。
*   **數據格式**：繳庫總量 `總:{值}`；生產天數為純數字；無效支撐天數顯示為 `-`。
*   **RWD**：手機端自動切換為卡片模式，利用 `data-label` 動態標註標籤。

### C. 互動特色
*   **一鍵清除**：搜尋框內置醒目的「紅色加粗 X」按鈕。
*   **點擊跳轉**：表格內 POY 批號可點擊觸發分析，並平滑跳轉至頂部。

## 4. 檔案結構
*   `reality_analyzer.py`：後端分析、清洗與支撐天數 Injection 引擎。
*   `generate_dashboard_v3.py`：Professional 版渲染器，負責互動 JS、對齊優化與 RWD。
*   `production_dashboard.html`：最終戰情室成品檔案。
