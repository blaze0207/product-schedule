# 撚二科生產戰情室 4.0 旗艦品質版技術手冊 (GEMINI4.0.md)

本文件定義 4.0 「旗艦品質版」的最終架構與技術邏輯，整合了精準狀態判斷、高效能數據引擎、S 系列機台專屬邏輯以及視覺化戰情室。

## 1. 系統架構 (System Architecture)

### 1.1 `RealityLogAnalyzer` (高效能數據引擎 - reality_analyzer.py)
*   **單次掃描技術 (Single-Pass Engine)**：整合產量累計與任務處理於單一迴圈，極大化大數據處理效能。
*   **資料時間落差補償 (Stability Reference Logic)**：
    *   **雙重日期判定**：自動識別 Excel 中的所有不重複日期。
    *   **狀態基準日 (Status Reference Date)**：強制選取「倒數第二個日期」作為網頁顯示與機台狀態運作的基準，徹底解決新舊班交接時（如早班尚未填寫完畢）導致的「空值誤判為停機」問題。
    *   **產量累計基準**：維持使用「絕對最新日期」進行產量累計，確保庫存進度與 KPI 數據不漏計。
*   **智慧庫存累計**：自動計算庫存表日期之後的現場產量，補足資訊落差。
*   **精準狀態判定 (Precision Status Mapping)**：
    *   **一般機台**：讀取生產資訊「夜班」X 欄 (L側) 與 Y 欄 (R側)。數字表運行，空值表停機，文字表特殊狀態。
    *   **S 系列機台**：讀取 AT 欄位 (Index 45)。
        *   **S1/S2**：專屬 Sec 轉換邏輯，自動計算「X Sec 生產中, (12-X) Sec 停機」。
*   **結構適應邏輯**：
    *   **單/雙列適應**：自動識別機台在 Excel 中佔用的列數，精確對接狀態與庫存。
    *   **名稱映射**：自動將 `S01/S02` 轉換為標準 `S1/S2`。

### 1.2 `DashboardRenderer` (渲染層 - generate_dashboard_v3.py)
*   **動態批號前綴**：針對運行中的一般機台自動附加「生產中:」字樣；已完工與 S2 機台則保持簡潔原始顯示。
*   **S 系列專屬設計**：
    *   **顯示優化**：隱藏 S1/S2 的 (L+R)側邊標記。
    *   **台數統計加權**：
        *   **S1**：≦ 6SEC 計為 0.5 台，> 6SEC 計為 1.0 台。
        *   **S2**：完全排除於運行/停機台數加總。
*   **多重連動篩選**：工程師、狀態按鈕與模糊搜尋框的三重過濾機制。
*   **自動偵測更新**：網頁端每 60 秒自動比對伺服器版本並重新整理。
*   **POY 支撐天數分析**：整合庫存與當前機台平均日耗，動態計算剩餘支撐天數。

### 1.3 `ExportEngine` (美化匯出 - export_cleaned_plan.py)
*   **自動清洗**：剔除 M4(CW)、V 欄位及其他非生產類雜訊。
*   **旗艦設計**：深藍配色、中文字自適應寬度、凍結首行，生成專業版 Excel (`*_clear.xlsx`)。

## 2. 關鍵維護與同步邏輯 (Maintenance Logic)

*   **自動化流程**：透過 `更新生產進度表.bat` 執行「Pull -> 分析 -> 生成 -> Commit -> Push」一鍵同步。
*   **全目錄同步**：所有原始 Excel 資料與程式腳本均納入 Git 管理，確保異地辦公之環境一致性。
*   **程式碼清理 (Cleanup)**：已移除所有過期、冗餘的迴圈與中間變數，保持專案極度精簡。
