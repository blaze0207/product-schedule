# 撚二科生產戰情室 5.0 旗艦品質版技術手冊 (GEMINI_5.0.md)

本文件定義 5.0 「旗艦品質版」的最終架構與技術邏輯，延續 4.0 的高效引擎，並新增了密碼保護、更嚴謹的 S1 判定邏輯、以及深化的 POY 原料控管分析。

## 1. 系統架構 (System Architecture)

### 1.1 `RealityLogAnalyzer` (數據核心 - reality_analyzer.py)
*   **單次掃描技術 (Single-Pass Engine)**：維持極大化效能，同時處理產量累計與狀態判定。
*   **穩定基準日邏輯 (Stability Reference Logic)**：
    *   **狀態基準日 (Status Reference Date)**：預設選取不重複日期中的「倒數第二個日期」，避上班交接期數據未填完之誤判。
    *   **產量累計基準**：使用「絕對最新日期」，確保每一噸的產量都納入累計計算。
*   **精準狀態判定 (Precision Status Mapping)**：
    *   **一般機台**：
        *   **運行中**：偵測到數字，或 AT 欄位標註「全」台。
        *   **停機**：空值、標註「停機」或批號為「---」。
    *   **S 系列機台 (方案 B 邏輯)**：
        *   **優先權判斷**：批號為「---」或標註「停機」關鍵字時，**絕對優先判定為停機**，並徹底忽略且移除 AT 欄位中的 `SEC` 殘留資訊。
        *   **運行描述**：非停機狀態下，顯示「X Sec 生產中, (12-X) Sec 停機」，與 S2 保持顯示一致。

### 1.2 `DashboardRenderer` (視覺化層 - generate_dashboard_v3.py)
*   **安全防護 (Security Enhancement)**：
    *   **網頁密碼鎖**：新增進入網頁時的密碼驗證（密碼：`fencdty`）。
    *   **智慧登入**：採用 `sessionStorage`，同分頁只需登入一次。
*   **POY 全場日耗分析 (方案 A 優化)**：
    *   **KPI 大型化**：POY 分析面板新增「全場日耗總計 (T/Day)」，將該批號在全場所有機台的每日消耗量加總並醒目顯示，便於原料控管與領料調度。
*   **統計加權與燈號連動**：
    *   **S1**：運行權重（≦6SEC 計 0.5 台，>6SEC 計 1.0 台），若批號為「---」則自動歸零計 0 台。
    *   **燈號邏輯**：強化關鍵字偵測（停機、了機、改紡等），一旦觸發，燈號強制顯示為 **🔴 停機**。
*   **UI/UX 改進**：
    *   **回到頂部**：新增浮動按鈕，方便快速導覽長表格。
    *   **自適應面板**：優化 POY 分析面板的佈局，確保多項 KPI 同時顯示時不擁擠。

### 1.3 `ExportEngine` (美化匯出 - export_cleaned_plan.py)
*   **清洗邏輯**：維持 4.0 的高效過濾，確保匯出的 `*_clear.xlsx` 乾淨且專業。

## 2. 關鍵維護與同步邏輯 (Maintenance Logic)

*   **自動化指令**：`更新生產進度表.bat` 持續負責同步、分析與生成。
*   **判定關鍵字**：內建 `status_keywords` 列表，包含「停機、了機、改紡、檢修、待料...」等。

*   **配色體系**：`Inter` 字體與深色主題，核心數據 (KPI) 使用高對比顯示。
